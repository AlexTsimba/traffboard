### **📚 Four Documentation Types**

#### **🎓 [Tutorials](tutorial/)**
*Learning-oriented guides*
- **Purpose**: Build confidence through hands-on practice
- **For**: Newcomers and those learning new features
- **Style**: Step-by-step, practical exercises
- **Start here**: [Getting Started](tutorial/getting-started.md)

#### **🛠️ [How-To Guides](how-to/)**  
*Problem-oriented guides*
- **Purpose**: Solve specific real-world problems
- **For**: Developers with some TraffBoard experience
- **Style**: Goal-oriented, practical instructions
- **Popular**: [Development Setup](how-to/development/setup-dev-env.md)

#### **📖 [Reference](reference/)**
*Information-oriented documentation*
- **Purpose**: Provide comprehensive technical details
- **For**: Developers needing exact specifications
- **Style**: Exhaustive, accurate, structured
- **Essentials**: [API Docs](reference/api/), [Components](reference/components/)

#### **🧠 [Explanation](explanation/)**
*Understanding-oriented documentation*
- **Purpose**: Clarify concepts and design decisions  
- **For**: Anyone wanting deeper understanding
- **Style**: Discursive, contextual, illuminating
- **Key topics**: [Architecture](explanation/architecture/), [Patterns](explanation/patterns/)
