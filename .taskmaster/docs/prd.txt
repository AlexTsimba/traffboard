# TraffBoard Universal Filter System with DEP2COST Implementation

## Product Overview
Build a production-ready cohort analysis report with DEP2COST metric and universal attribution-aware filter system that serves as foundation for all future reports (Financial, Marketing, etc.).

## Core Requirements

### Universal Attribution System
- Attribution dropdown in General Filters tab: "FTD Date" vs "Registration Date" 
- Attribution affects all data grouping across any report type
- Dynamic SQL GROUP BY field selection (signUpDate vs firstDepositDate)
- Foundation for Financial and Marketing reports

### Filter System Enhancement
- Fix broken cascading filters: Partner → Campaign → Country
- Add third "Params" tab with tagOs, tagSource, tagSub2, tagWebId filters
- Universal API endpoint: /api/universal/filter-options
- Real-time cascading with attribution support

### DEP2COST Implementation  
- Replace mock data with real calculations
- Formula: sum(depositsSum) / sum(fixedPerPlayer) per breakpoint
- Attribution-based cohort grouping
- Percentage formatting (e.g., "20.5%")
- Support day/week breakpoints: [1,3,5,7,14,17,21,24,27,30] and [7,14,21,28,35,42]

### Data Pipeline Integration
- Uncomment and fix existing CohortProcessor
- PostgreSQL-Arquero pipeline with attribution support  
- Performance target: sub-3 second query times
- Support all filter types through universal API

## Success Criteria
- Working DEP2COST tab with real data and attribution switching
- Universal filter system ready for Financial/Marketing reports
- All cascading filters functional
- Zero TypeScript errors, all tests pass
- Demo-ready within 3 days

- Current: 80% implementation complete with existing UI components
- Stack: Next.js 15 + React 19 + TypeScript + PostgreSQL + Prisma
